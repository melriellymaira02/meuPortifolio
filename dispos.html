    Dispositivos móveis React-Native O react-native é o filho do react o react-native foi desenvolvido pelo o Facebook ele é renderizado e escrito com JavaScript é disponível em diversas plataformas Estrutura básica do React-Native import React from "react"; import {View} from "react-native"; export default function App(){ return( ); } Simplificando ele está importando o React da biblioteca react, depois ele vai o componente View da biblioteca do react-native e então está importando uma função padrão chamada App para ele retornar a renderização(“mostrar”) dos componentes para o nosso app. Componentes Componentes são os comandos que renderizam os itens que desejamos para o App. Todos esses componentes têm algum atributo para modificá-lo do jeito que deseja independente se for um atributo global, ou seja, os atributos servem para qualquer componente (nesse caso iremos aprender os essenciais) e alguns tem métodos. Android IOS Componente* Componente* Obs:Os que não tiverem com marcador em cima é para ambos os Sistemas Operacionais View Faz aparecer o item que estiver dentro dela na tela geralmente usado como componente pai (o componente vira uma “caixa” que sustenta todos os componentes dentro dele) SafeAreaView É o View só que agora no IOS o view fica em cima da barra de notificações o SafeAreaView não fica em cima fica no local da aplicação TextInput Coloca uma caixa para o usuário digitar nela Atributos placeholder (vai colocar um exemplo antes do usuário digitar. Tipo STRING entre aspas) keyboardType (vai trocar o teclado virtual do aparelho. Respostas do Tipo STRING: 'default', 'email-address', 'numeric', 'phone-pad', 'ascii-capable', 'numbers-and-punctuation', 'url', 'number-pad', 'name-phone-pad', 'decimal-pad', 'twitter', 'web-search', 'visible-password') textAlign (vai alinhar o que o usuário vai digitar. Respostas do Tipo STRING: 'left', 'center', 'right') Button Cria um botão para ele funcionar corretamente precisa desses dois atributos onPress (vai falar o que botão vai fazer depois de clicá-lo. Tipo STRING coloque o evento depois dos dois pontos desse atributo) title (vai colocar o título do botão. Tipo STRING o título tem que estar entre aspas) Image Coloca a imagem que escolher esse componente precisa ter o tamanho da altura (height) e largura (width) na linguagem da folha de cascata “CSS” além de que você precisa falar qual método você vai usar EX: se usar o método uri você tem que achar o link da sua imagem FlatList Antes de usar esse componente você precisa criar uma lista em uma const antes de exportar a renderização da sua tela. Esse componente cria uma “caixa” para cada item que estiver dentro de sua constante array esse componente precisa de dois atributos renderItem (renderizará o item da lista precisa ser escrito entre aspas) data (você vai falar qual o nome da lista para ele localizar ela e pegar seu item) React-Navigation Essa biblioteca é o que faz a transição de telas. Essa biblioteca tem que ser “instalada” em seu projeto em sua dependência, ou seja, você escreve as dependências necessárias para usar essa biblioteca antes de poder instalar ela no seu app (quando acabar seu projeto). O uso desse item consiste que você crie uma função que renderizará na outra tela Uso de Bibliotecas Antes de mais nada você precisa entender o que sua biblioteca vai fazer (por exemplo para colocar uma fonte fora do padrão você tem que saber qual biblioteca ela está localizada como o Google fonts) então instalar as dependências da biblioteca para importar o recurso que deseja.
