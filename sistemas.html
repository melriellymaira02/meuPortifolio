EX: email🡪fnbhm aqui adicionamos +1 com base no alfabeto ou seja depois do “e” é o “f” e assim por diante Criptografia assimétrica é uma criptografia que contém duas chaves uma pública e privada. A primeira chave é uma chave pública usada para criptografar uma mensagem e a segunda é uma chave privada utilizada para descriptografá-la. Criptografia simétrica é uma criptografia que tem somente uma chave como o exemplo acima. Hashing transforma os dados para algo fixo que sequer sejam reconhecíveis. Usado em criação de assinaturas digitais e análise e verificação da validade e integridade de arquivos digitais. Protocolos Algumas delas são DES, 3DES, AES, IDEA, RC4, TLS e SSL. Importante: existem criptografias que não tem chaves chamadas de HASH. Tipos Simétrica, Assimétrica e Hashing. Cookies São fragmentos de dados que o navegador envia ao usuário para salvar informações do usuário e pode usar essas informações. Eles têm um tempo de vida, ou seja, após algum certo tempo (determinado pelo desenvolvedor) o cookie apagará suas informações. Session São os cookies só que eles são apagados quando você fecha se browser (navegador) HTTP (Hyper Text Transport Protocol) É um protocolo da camada de aplicação para transmissão de documentos hipermídia como o HTML, ele comunica com os navegadores web e servidores web. HTTP status: Respostas de informação (100-199), Respostas de sucesso (200-299), Redirecionamentos (300-399), Erros do cliente (400-499) e Erros do servidor (500-599). Estrutura de três Camadas (3 Tier Architecture) É uma arquitetura que é utilizada para o programa conversar com o banco de dados sem quaisquer “aberturas” fazendo que seja mais difícil hackers acessar o banco de dados e comprometer os dados. Primeira camada Chamada de camada de apresentação ela é o layout do programa exibindo informações na parte de interface gráfica. Segunda camada Chamada de camada de aplicação nesta camada os dados obtidos da camada de apresentação são processados Terceira camada Camada de dados nesta camada as informações da camada de aplicação são armazenadas e gerenciadas Obs: Existem aplicativos que usam mais que somente três camadas, mas, não é recomendado pois, com isso o programa demora mais além de que por ter mais de três camadas qualquer erro acaba criando uma abertura no programa. Operação CRUD Método HTTP Crate (crie) Post (crie) Read (leia) Get (leia) Update (atualize) Patch/Put (atualize) Delete (exclua) Delete (exclua) Obs: Precisa estar em um JSON para realizar alguma operação CRUD e com isso ele manda um HTTP status. HTTPS É o HTTP só que com segurança por isso o “s” de secure. Se obtém esse secure após o site ter um certificado SSL (emitido por uma autoridade de certificação).
